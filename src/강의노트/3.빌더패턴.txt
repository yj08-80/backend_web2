[ 빌더패턴 ]
    1. 정의 : Lombok 라이브러리에서 제공하는 빌더 패턴
    2. 목적 : 복잡한 객체 생성을 간단하고 직관적으로 표현
    3. 장점
        1) 매개변수가 많을 때 유용( 순서규칙 없음 )
        2) 필요한 매개변수만 선택적으로 초기화

    4. 주요 메소드
        1) .builder() : 빌더 객체 시작점 *static
        2) .속성명(초기값) : 지정한 속성명에 초기값 대입
        3) .build() : 빌더 객체 끝지점

    5. 사용법
        1) 빌더 패턴을 사용할 클래스 위에
            @Builder
            @NoArgsConstructor @AllArgsConstructor
        2) 빌더패턴 사용
            클래스명 변수명 = new 생성자명( 값1, 값2 );
                                vs
            [빌더패턴] 클래스명 변수명 = 클래스명.builder().속성명(값1).속성명(값2).build();
